# Agent可读参考资料

本目录存储项目中使用的外部工具、设计系统或框架的紧凑、Agent可读的参考资料文件。

## 目的

在编码会话期间，Agent无法可靠地浏览网页或访问外部文档。通过在此处保存关键参考资料，Agent可以在不依赖聊天上下文或人类记忆的情况下发现和使用它们。

## 存放内容

- 设计系统令牌和组件API
- 部署平台规则（例如，nixpacks、Dockerfile约定）
- 特定语言的工具链规则（例如，uv、pip、cargo）
- API模式或协议定义
- 第三方库使用模式

## 命名约定

使用模式：`{工具名称}-llms.txt`

示例：
- `design-system-reference-llms.txt`
- `nixpacks-llms.txt`
- `uv-llms.txt`

## 维护

- 保持文件紧凑和聚焦（每个文件不超过500行）
- 当上游工具变更时刷新资料
- 删除项目中不再使用的参考资料
